# 微电流美容仪 - 通信协议文档

| 版本 | 日期 | 主要改动 |
|------|------|----------|
| V1.0 | 2026-05-08 | 初始版本，汇总蓝牙串口通信协议 |

---

## 一、概述

设备与手机APP之间通过 **UART1（9600bps）** 经蓝牙模块进行通信。通信采用 **主从应答模式**，APP为主机，设备为从机。

---

## 二、帧格式

所有数据帧采用统一格式：

```
0x55 0xAA [len] [idH] [idL] [type] [state] [func] [para1] ... [paraN] [crcH] [crcL]
```

| 字节位置 | 字段 | 长度 | 说明 |
|----------|------|------|------|
| 1 | HEAD1 | 1B | 帧头1，固定 `0x55` |
| 2 | HEAD2 | 1B | 帧头2，固定 `0xAA` |
| 3 | len | 1B | 数据域长度（从idH到paraN的总字节数） |
| 4 | idH | 1B | 设备ID高字节，固定 `0x00` |
| 5 | idL | 1B | 设备ID低字节，固定 `0xA6` |
| 6 | type | 1B | 帧类型（见下方类型定义） |
| 7 | state | 1B | 帧状态（见下方状态定义） |
| 8 | func | 1B | 功能码 |
| 9 ~ (8+N) | para1~paraN | 0~3B | 参数列表（最大3字节） |
| (9+N) | crcH | 1B | CRC16-MODBUS 校验高字节 |
| (10+N) | crcL | 1B | CRC16-MODBUS 校验低字节 |

**len 计算公式**: `len = 5 + N`（idH + idL + type + state + func + 参数字节数）

---

## 三、帧类型（type）

| 值 | 宏定义 | 方向 | 说明 |
|----|--------|------|------|
| `0x31` | FRAME_TYPE_ACT | APP → 设备 | 动作命令（启动/停机/升级等） |
| `0x32` | FRAME_TYPE_SET | APP → 设备 | 参数设置（制冷档位等） |
| `0x33` | FRAME_TYPE_GET | APP → 设备 | 参数读取（版本号/温度等） |
| `0x66` | FRAME_TYPE_POST | 设备 → APP | 主动上报 |

---

## 四、帧状态（state）

| 值 | 宏定义 | 说明 |
|----|--------|------|
| `0x00` | FRAME_STATE_ERR | 校验出错（从机回复） |
| `0x01` | FRAME_STATE_ACK | 应答成功（从机回复） |
| `0x02` | FRAME_STATE_ASK | 上位机请求（主机发送） |

---

## 五、指令集

### 5.1 动作命令（type = 0x31）

#### 5.1.1 启动/停机控制（func = 0x01）

| 方向 | 说明 |
|------|------|
| APP → 设备 | 控制设备启动工作或停机 |

**请求帧**:
```
55 AA 08 00 A6 31 02 01 [run] [crcH] [crcL]
```

| 参数 | 值 | 说明 |
|------|----|------|
| run | `0x01` | 启动（使能PID制冷） |
| run | `0x00` | 停机（关闭PID制冷） |

**应答帧**:
```
55 AA 08 00 A6 31 01 01 [run] [crcH] [crcL]
```

**行为说明**:
- 启动 (`run=1`): 使能PID制冷控制，系统开始制冷工作
- 停机 (`run=0`): 关闭PID制冷控制，关闭CCP0 PWM输出

---

#### 5.1.2 进入老化模式（func = 0x05）

| 方向 | 说明 |
|------|------|
| APP → 设备 | 设备进入老化测试模式 |

**前提条件**: 需先开机（PowerOn=1）且已启动（WorkStart=1）

**请求帧**:
```
55 AA 07 00 A6 31 02 05 [crcH] [crcL]
```

**应答帧**:
```
55 AA 07 00 A6 31 01 05 [crcH] [crcL]
```

---

#### 5.1.3 远程升级（func = 0xAA）

| 方向 | 说明 |
|------|------|
| APP → 设备 | 触发设备固件升级，设备将复位 |

**请求帧**:
```
55 AA 07 00 A6 31 02 AA [crcH] [crcL]
```

**应答帧**:
```
55 AA 07 00 A6 31 01 AA [crcH] [crcL]
```

**行为说明**: 收到此指令后，设备写入升级标志位并执行MCU复位。

---

### 5.2 参数设置（type = 0x32）

#### 5.2.1 设置制冷档位（func = 0x0C）

| 方向 | 说明 |
|------|------|
| APP → 设备 | 设置制冷目标温度档位 |

**请求帧**:
```
55 AA 08 00 A6 32 02 0C [level] [crcH] [crcL]
```

| 档位 (level) | 目标温度 | 说明 |
|--------------|----------|------|
| `0x01` | 20.0°C | 最高档（温度最高，制冷最弱） |
| `0x02` | 17.0°C | 二档 |
| `0x03` | 13.0°C | 三档 |
| `0x04` | 8.0°C | 四档 |
| `0x05` | 5.0°C | 最低档（温度最低，制冷最强） |

**应答帧**:
```
55 AA 08 00 A6 32 01 0C [level] [crcH] [crcL]
```

---

### 5.3 参数读取（type = 0x33）

#### 5.3.1 读取固件版本号（func = 0x40）

| 方向 | 说明 |
|------|------|
| APP → 设备 | 读取当前固件版本 |

**请求帧**:
```
55 AA 07 00 A6 33 02 40 [crcH] [crcL]
```

**应答帧**:
```
55 AA 09 00 A6 33 01 40 [major] [minor] [crcH] [crcL]
```

| 参数 | 说明 |
|------|------|
| major | 主版本号（当前: 2） |
| minor | 次版本号（当前: 1） |

---

#### 5.3.2 读取当前温度（func = 0x41）

| 方向 | 说明 |
|------|------|
| APP → 设备 | 读取NTC传感器当前温度 |

**请求帧**:
```
55 AA 07 00 A6 33 02 41 [crcH] [crcL]
```

**应答帧**:
```
55 AA 09 00 A6 33 01 41 [tempH] [tempL] [crcH] [crcL]
```

| 参数 | 说明 |
|------|------|
| tempH | 温度值高字节 |
| tempL | 温度值低字节 |
| 温度值 | 有符号整数，单位 0.1°C（例: `0x0069` = 105 = 10.5°C） |

## 六、CRC16-MODBUS 校验

- **算法**: CRC16-MODBUS
- **多项式**: `0xA001`（反转 `0x8005`）
- **初始值**: `0xFFFF`
- **计算范围**: 从 `len` 字节开始到 `paraN` 字节结束（不含帧头 0x55 0xAA 和 CRC 本身）
- **字节序**: CRC高字节在前，低字节在后

```c
INT16U CrcCalc_Crc16Modbus(INT8U *dat, INT8U len)
{
    INT16U CRC_index = 0xFFFF;
    INT16U buffer;
    INT8U i, j;
    for (i = 0; i < len; i++) {
        buffer = dat[i];
        CRC_index ^= buffer;
        for (j = 0; j < 8; j++) {
            if (CRC_index & 0x0001) {
                CRC_index >>= 1;
                CRC_index ^= 0xA001;
            } else {
                CRC_index >>= 1;
            }
        }
    }
    return CRC_index;
}
```

## 七、通信时序

### 7.1 正常请求-应答流程

```
APP (主机)                         设备 (从机)
   |                                   |
   |--- 请求帧 (state=0x02) --------->|
   |                                   |--- 解析指令
   |                                   |--- 执行操作
   |<-- 应答帧 (state=0x01) ----------|
   |                                   |
```

### 7.2 校验失败流程

```
APP (主机)                         设备 (从机)
   |                                   |
   |--- 请求帧 (CRC错误) ----------->|
   |                                   |--- CRC校验失败
   |<-- 错误帧 (state=0x00) ----------|
   |                                   |
```

### 7.3 蓝牙连接状态管理

- 蓝牙断开时，设备拒绝处理所有指令（`UART1_RecvDecode` 重置解码状态机）
- 蓝牙断开时，设备自动关闭PID制冷（安全保护）
