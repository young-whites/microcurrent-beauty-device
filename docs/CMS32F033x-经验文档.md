> 维护方式：每次踩坑后追加记录

## 一、CCP 模块（Capture/Compare/PWM）

### 1.1 CCP 使能是双层结构

CMS32F033 的 CCP 模块使能分为**两步**，缺一不可：

| 函数 | 作用 | 寄存器 | 含义 |
|------|------|--------|------|
| `CCP_Start(ch)` | 设置 `CCP->CONx` 的 EN 位 | CONx | 使能 CCP 模块（模块级） |
| `CCP_EnableRun(ch)` | 设置 `CCP->RUN` | RUN | 启动定时器计数（通道级） |

**只调 `CCP_Start` 没有输出**，只调 `CCP_EnableRun` 也没有输出，**必须两个都调**。

```c
// 正确示例
SYS_SET_IOCFG(IOP07CFG, SYS_IOCFG_P07_CCP1B);
SYS_EnablePeripheralClk(SYS_CLK_CCP_MSK);
CCP_ConfigCLK(CCP1, CCP_CLK_DIV_1, CCP_RELOAD_CCPLOAD, 3000);
CCP_EnablePWMMode(CCP1);
CCP_ConfigDutyScale(CCP1, CCPxB, 0);
CCP_Start(CCP1);        // 模块级使能
CCP_EnableRun(CCP1);    // 通道级使能（启动计数器）
```

**踩坑后果**：模块开了但定时器没跑，PWM 无输出且**不报错**，非常隐蔽。

### 1.2 CCP_Stop 后重启必须重新 CCP_Start

`CCP_Stop()` 会清除模块使能位（CONx 的 EN 位），再次输出前**必须**重新调用 `CCP_Start()`。

```c
// 停止后重启的正确写法
void EnergyStart(void) {
    CCP_Start(CCP1);        // 重新使能模块
    CCP_EnableRun(CCP1);    // 重新启动计数
    CCP_ConfigCompare(CCP1, CCPxB, target_duty);
}
```

**踩坑后果**：调了 `Energy_Off()` 再调 `EnergyStart()`，忘记重设 CCP_Start，输出永久丢失。

### 1.3 通道参数 xA / xB 必须对应正确引脚

CMS32F033 的 CCP 通道与 GPIO 映射是固定的：

| 引脚 | CCP 通道 | 备注 |
|------|----------|------|
| P30 | CCP0A | xA 通道 |
| P07 | CCP1B | xB 通道 |

`CCP_ConfigDutyScale()`、`CCP_ConfigCompare()` 等函数的通道参数必须用正确的 `CCPxA` 或 `CCPxB`，写错通道 = 写错寄存器 = 另一个引脚有输出，目标引脚没有。

### 1.4 不同 CCP 实例之间互不影响但共享时钟

- CCP0 和 CCP1 是**独立模块**，各有自己的 CON/RUN/CMP/LOAD 寄存器
- 但它们共享 `SYS_CLK_CCP` 时钟门控，`SYS_EnablePeripheralClk(SYS_CLK_CCP_MSK)` 开一次即可
- 初始化 CCP1 时**不要碰 CCP0 的寄存器**，反之亦然

### 1.5 GPIO 复用必须在启动函数中重复设置

`SYS_SET_IOCFG()` 写的是 IOCFG 寄存器，如果系统中有其他初始化函数运行在后面，可能会覆盖复用配置。

**安全做法**：在 Init 和每次启动/重启动能输出时都设置一次 IOCFG。

```c
void EnergyStart(void) {
    SYS_SET_IOCFG(IOP07CFG, SYS_IOCFG_P07_CCP1B);  // 防止被覆盖
    CCP_Start(CCP1);
    CCP_EnableRun(CCP1);
    // ...
}
```

## 二、EPWM 模块

### 2.1 EPWM 通道时钟分频注意硬件除法器

EPWM1 的时钟计算与其他通道不同（使用硬件除法器 HWDIV），配置时需注意：

```c
// EPWM1 特殊处理
EPWM_ConfigChannelDiv(1, Div[1]);
HWDIV_EnableUnsingedMode();
Period = HWDIV_Div(SystemAPBClock, Freq[1]);
Period = HWDIV_Div(Period, Div[1]);
EPWM_ConfigChannelPeriod(1, Period >> 1);

// 其他通道（EPWM0/2/3/4/5）
EPWM_ConfigChannelDiv(i, Div[i]);
Period = SystemAPBClock / Freq[i] / Div[i];
EPWM_ConfigChannelPeriod(i, Period >> 1);
```

### 2.2 EPWM 必须调用 EPWM_Start 才有输出

`EPWM_Config_Independent_Mode()` 只配置参数，不会自动启动。配置完成后必须调用 `EPWM_Start(mask)` 启动输出。

### 2.3 EPWM 中断标志必须清除

EPWM 的上/下比较点中断触发后，必须在 ISR 中清除对应标志，否则会反复进入中断导致系统锁死。

```c
void EPWM_IRQHandler(void) {
    if (EPWM_GetUpCmpIntFlag(EPWM_CH_2_MSK))
        EPWM_ClearUpCmpIntFlag(EPWM_CH_2_MSK);
    if (EPWM_GetDownCmpIntFlag(EPWM_CH_2_MSK))
        EPWM_ClearDownCmpIntFlag(EPWM_CH_2_MSK);
    // 其他通道同理
}
```

## 三、Timer 模块（TMR0/TMR1）

### 3.1 TMR0 的 reload 值直接影响中断频率

`Pclk = 3MHz` 时，reload 值计算公式：

```
中断频率 = Pclk / reload
reload = 3 (错误!) → 中断频率 = 1MHz → 系统锁死
reload = 3000 → 中断频率 = 1kHz → 正常
```

**踩坑后果**：reload 写得太小，中断频率过高，CPU 100% 被中断占用，系统死机，串口无响应。

### 3.2 TMR0 需要单独开启外设时钟

```c
SYS_EnablePeripheralClk(SYS_CLK_TIMER01_MSK);
TMR_EnableRun(TMR0);
```

## 四、通用调试经验

### 4.1 PWM 无输出排查清单

按以下顺序检查，覆盖 90% 的问题：

| 序号 | 检查项 | 命令/方法 |
|------|--------|-----------|
| 1 | GPIO 复用是否设置 | 查 IOCFG 寄存器值 |
| 2 | 外设时钟是否开启 | 查 SYS CLK 寄存器 |
| 3 | 模块是否使能（CCP_Start / EPWM_Start） | 查 CON 寄存器 EN 位 |
| 4 | 定时器是否运行（CCP_EnableRun） | 查 RUN 寄存器 |
| 5 | 通道参数是否正确（xA vs xB） | 核对 datasheet 引脚映射 |
| 6 | 占空比是否 > 0 | 查 CMP 寄存器 |
| 7 | 有没有被其他初始化覆盖 | 用调试器在 Init 尾部断点读寄存器 |

### 4.2 系统锁死常见原因

| 现象 | 最可能原因 |
|------|-----------|
| 上电后立即死机 | Timer reload 值太小（中断频率过高） |
| 运行一段时间后死机 | 中断标志未清除（嵌套中断堆积） |
| 某个功能启动后死机 | NVIC 优先级冲突或中断未注册 |

### 4.3 初始化顺序原则

```
1. GPIO 复用配置（IOCFG）
2. 外设时钟使能（SYS_EnablePeripheralClk）
3. 外设参数配置（频率、模式、占空比）
4. 外设使能（CCP_Start / EPWM_Start）
5. 计数器启动（CCP_EnableRun）
```

**顺序错误**可能导致配置写入无效或被覆盖。
