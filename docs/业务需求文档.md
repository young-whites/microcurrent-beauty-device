## 一、产品概述

本产品为手持式微电流美容仪（Dr-05），通过蓝牙连接手机 APP 进行控制，核心功能包括微电流导入和冷敷护理。设备采用 CMS32F033x 单片机作为主控，通过 UART1 连接 BLE 模块与手机 APP 通信。

### 3.3 APP 通信协议

**当前实现状态**：🟡 框架已搭建，指令集不完整

#### 3.3.1 协议帧格式

```
帧头(2B) + 长度(1B) + 数据(NB) + CRC16(2B)
0x55 0xAA [len] [idH] [idL] [type] [state] [func] [para1]...[paraN] [crcH] [crcL]
```

- **帧头**：0x55 0xAA
- **设备 ID**：0x00A6
- **CRC**：CRC16-Modbus
- **超时**：2000ms

#### 3.3.2 帧类型

| 类型码 | 含义 |
|--------|------|
| 0x31 | 动作命令 (ACT) |
| 0x32 | 参数设置 (SET) |
| 0x33 | 参数获取 (GET) |
| 0x66 | 主动上报 (POST) |

### 3.7 离子输出控制 (SN74HC21D 半桥驱动)

**当前实现状态**：🟢 驱动已重构

#### 硬件架构

采用双半桥结构，上半桥输出 ION+（正离子），下半桥输出 ION-（负离子），通过 SN74HC21D 双4输入与门实现逻辑使能控制。

| SN74HC21D引脚 | MCU引脚 | 功能配置 | 用途 |
|---|---|---|---|
| 9, 5 | P10 | GPIO | 使能控制 |
| 2, 12 | P31 | GPIO | 使能控制(上桥) |
| 10 | P06 | EPWM3 | 上半桥PWM输出 |
| 13 | P40 | GPIO | 使能控制 |
| 1 | P12 | GPIO | 使能控制(下桥) |
| 4 | P00 | EPWM2 | 下半桥PWM输出 |

#### API 接口

**基础控制：**

| 函数 | 说明 |
|------|------|
| `SN74HC21D_Init()` | 初始化所有GPIO+EPWM外设 |
| `SN74HC21D_OpenUpper()` | 启动上半桥(ION+)，使能上桥GPIO+EPWM3输出 |
| `SN74HC21D_OpenLower()` | 启动下半桥(ION-)，使能下桥GPIO+EPWM2输出 |
| `SN74HC21D_StopAll()` | 紧急停止，关闭所有半桥输出 |
| `SN74HC21D_DeadTime()` | 死区延时，防止上下半桥同时导通 |

**占空比调节：**

| 函数 | 说明 |
|------|------|
| `SN74HC21D_SetDutyUpper(uint8_t duty)` | 设置上半桥占空比(0~100%) |
| `SN74HC21D_SetDutyLower(uint8_t duty)` | 设置下半桥占空比(0~100%) |

**正弦波生成：**

| 函数 | 说明 |
|------|------|
| `SN74HC21D_SineWaveStep(uint8_t step)` | 执行一步正弦波(0~15) |
| `SN74HC21D_SineWaveEnable(uint16_t freq_hz)` | 启用正弦波自动生成(TMR0驱动) |
| `SN74HC21D_SineWaveDisable()` | 停用正弦波自动生成 |

#### EPWM 动态接口

| 函数 | 说明 |
|------|------|
| `EPWM_SetDutyPercent(uint8_t ch_idx, uint8_t duty)` | 动态调整指定通道占空比 |
| `EPWM_ReadPeriod(uint8_t ch_idx)` | 读取指定通道周期值 |

#### 正弦波生成机制

- 采用 **16段/周期** 的阶梯正弦波方案
- 正半周(step 0~7)：使用上半桥(EPWM3)驱动，输出 ION+
- 负半周(step 8~15)：使用下半桥(EPWM2)驱动，输出 ION-
- 段间强制 **死区保护**，确保上下半桥不会同时导通
- 由 TMR0 定时器中断驱动自动步进，频率可配置

## 四、系统调度架构

### 4.1 时基调度 (Timer1 中断, 1ms 周期)

| 周期 | 执行内容 |
|------|----------|
| 1ms | LED_DrvScan(), BEEP_DrvScan(), APP_TimeOverScan() |
| 10ms | KEY_DrvScan(), Bluetooth_Scan() (仅开机状态) |
| 20ms | KEY_Scan() (按键事件处理) |
| 50ms | (预留) |
| 100ms | (预留) |
| 250ms | (预留) |
| 500ms | (预留) |
| 1s | (预留) |

### 4.1.1 EPWM 中断处理 (EPWM_IRQHandler)

| 中断源 | 优先级 | 处理内容 |
|--------|--------|----------|
| EPWM2/EPWM3 比较中断 | 3 | 清除上比较(CMPA)和下比较(CMPB)中断标志，维持 EPWM 正常运行 |

### 4.2 主循环流程

```
while(1):
  1. UART1_SendScan()     — 发送缓冲区→UART端口
  2. UART1_RecvScan()     — 接收缓冲区→协议解码
  3. SystemWorkStatusCheck() — 电源状态机
  4. IAP 升级判断
  5. 蓝牙连接状态检测
  6. 蓝牙断连→关闭所有输出
```

### 4.3 电源状态机

```
PowerKeyCnt=0 → 待机(关机)
  ↓ 按键短按
PowerKeyCnt=1 → 开机(蓝牙上电, UART切换, 功能就绪)
  ↓ 按键短按
PowerKeyCnt=2 → 关机(恢复待机)
```
