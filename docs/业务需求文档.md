> **生成日期**：2026-05-07  
> **来源**：基于 sourceCode 代码逆向拆解

---

## 一、产品概述

本产品为手持式微电流美容仪（Dr-05），通过蓝牙连接手机 APP 进行控制，核心功能包括 EMS 微电流导入和热敷护理。设备采用 CMS32F033x 单片机作为主控，通过 UART1 连接 BLE 模块与手机 APP 通信。

### 3.3 APP 通信协议

**当前实现状态**：🟡 框架已搭建，指令集不完整

#### 3.3.1 协议帧格式

```
帧头(2B) + 长度(1B) + 数据(NB) + CRC16(2B)
0x55 0xAA [len] [idH] [idL] [type] [state] [func] [para1]...[paraN] [crcH] [crcL]
```

- **帧头**：0x55 0xAA
- **设备 ID**：0x00A6
- **CRC**：CRC16-Modbus
- **超时**：2000ms

#### 3.3.2 帧类型

| 类型码 | 含义 |
|--------|------|
| 0x31 | 动作命令 (ACT) |
| 0x32 | 参数设置 (SET) |
| 0x33 | 参数获取 (GET) |
| 0x66 | 主动上报 (POST) |

## 四、系统调度架构

### 4.1 时基调度 (Timer1 中断, 1ms 周期)

| 周期 | 执行内容 |
|------|----------|
| 1ms | LED_DrvScan(), BEEP_DrvScan(), APP_TimeOverScan() |
| 10ms | KEY_DrvScan(), Bluetooth_Scan() (仅开机状态) |
| 20ms | KEY_Scan() (按键事件处理) |
| 50ms | (预留) |
| 100ms | (预留) |
| 250ms | (预留) |
| 500ms | (预留) |
| 1s | (预留) |

### 4.2 主循环流程

```
while(1):
  1. UART1_SendScan()     — 发送缓冲区→UART端口
  2. UART1_RecvScan()     — 接收缓冲区→协议解码
  3. SystemWorkStatusCheck() — 电源状态机
  4. IAP 升级判断
  5. 蓝牙连接状态检测
  6. 蓝牙断连→关闭所有输出
```

### 4.3 电源状态机

```
PowerKeyCnt=0 → 待机(关机)
  ↓ 按键短按
PowerKeyCnt=1 → 开机(蓝牙上电, UART切换, 功能就绪)
  ↓ 按键短按
PowerKeyCnt=2 → 关机(恢复待机)
```
